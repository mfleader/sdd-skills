When planning or generating tasks for the gap audit extension (specs/001-gap-audit-extension), include a task to create an example `gap-patterns.md` file. This file should demonstrate the structure defined in the spec assumptions: canonical name, checklist category, and trigger description per pattern. The example serves as a reference for implementers and users.

**Why:** The spec defines the format but a concrete example makes it unambiguous for anyone parsing or authoring the file.

**How to apply:** Add it as a deliverable in the plan or as a task during `/speckit.tasks`.
