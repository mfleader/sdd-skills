The brainstorm/ directory stores session notes from the superpowers:brainstorming skill. It's a checkpoint for resuming parked brainstorm sessions, not project documentation. Once ideas graduate to a spec, the brainstorm files are only useful if you want to revisit a parked thread later.

**Why:** Avoids confusion about whether brainstorm files are project artifacts or scratchwork.

**How to apply:** Don't treat brainstorm files as decision records or onboarding material. They're session state for the brainstorming skill.
